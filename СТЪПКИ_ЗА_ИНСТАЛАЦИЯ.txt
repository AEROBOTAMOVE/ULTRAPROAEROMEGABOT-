AERO BOT (злато 🥇 + сребро 🥈) — ИНСТАЛАЦИЯ ОТ НУЛАТА
Нов бот + нова група + ново PUBLIC repo. Стъпка по стъпка.
================================================================

──────────────────────────────────────────────
ЧАСТ 1 · НОВ ТЕЛЕГРАМ БОТ (2 мин)
──────────────────────────────────────────────
1.  В Телеграм отвори: @BotFather
2.  Напиши: /newbot
3.  Име (напр.: AERO Metals Bot)
4.  Username, ЗАДЪЛЖИТЕЛНО завършва на "bot" (напр.: aero_metals_bot)
5.  BotFather дава ТОКЕН → копирай го. Слага се САМО в GitHub
    Secret (стъпка 15) и на мен за проверката — никъде другаде!

──────────────────────────────────────────────
ЧАСТ 2 · НОВА ГРУПА (2 мин)
──────────────────────────────────────────────
6.  Създай група (напр.: "AERO СИГНАЛИ 🥇🥈")
7.  Добави бота в групата
8.  ⚠️ Направи бота АДМИН: групата → Administrators → Add admin → бота
    (без админ ботът не вижда групата и няма chat_id!)
9.  Пусни едно съобщение в групата ("тест")

──────────────────────────────────────────────
ЧАСТ 3 · CHAT ID (1 мин)
──────────────────────────────────────────────
10. НАЧИН А (най-лесно): прати ми токена в чата → аз изваждам
    chat_id, тествам връзката и ти казвам числото.
    НАЧИН Б (сам): отвори в браузъра (замени ТОКЕНА):
      https://api.telegram.org/botТВОЯ_ТОКЕН/getUpdates
    Търси "chat":{"id":-100....  → ОТРИЦАТЕЛНОТО дълго число
    (супергрупа = започва с -100...). Това е TELEGRAM_CHAT_ID.
    ⚠️ Ако направиш бота админ СЛЕД взимане на id → id-то се СМЕНЯ
    (групата става supergroup)! Първо админ, после id.

──────────────────────────────────────────────
ЧАСТ 4 · НОВО REPO (5 мин)
──────────────────────────────────────────────
11. github.com → "+" → New repository → име (напр. aero-metals)
    → избери PUBLIC (за неограничени минути и точен график!)
    → Create repository
    (Секретите са скрити и в Public — токенът е в безопасност.)

12. КАЧИ ФАЙЛОВЕТЕ: "uploading an existing file" → влачи тези 6:
      • live_bot.py
      • backtest_stats.json
      • requirements.txt
      • weekly_context.json
      • README.md
      • ПРИМЕРНА_СЕДМИЦА.txt
    ⚠️ БЕЗ .pyc файлове! Папката .github НЕ се качва с влачене —
    прави се на ръка в стъпка 13!
    → Commit changes

13. WORKFLOW (важно — точно така!):
    Add file → Create new file
    Име на файла, ТОЧНО (със слешовете):
        .github/workflows/aero-bot.yml
    Съдържание: отвори "workflow_за_копиране.txt" от папката
    с Notepad → Ctrl+A → Ctrl+C → залепи в GitHub → Commit changes

14. Ако пита в Actions → "I understand my workflows, enable them"

──────────────────────────────────────────────
ЧАСТ 5 · СЕКРЕТИ И ПРАВА (2 мин) — БЕЗ ТОВА НЕ РАБОТИ!
──────────────────────────────────────────────
15. Settings → Secrets and variables → Actions →
    New repository secret (двата поотделно):
      Name: TELEGRAM_TOKEN     Value: токена от стъпка 5
      Name: TELEGRAM_CHAT_ID   Value: числото от стъпка 10 (с минуса!)
    ⚠️ АКО СМЕНИШ бота/групата ПО-КЪСНО → ЗАДЪЛЖИТЕЛНО обнови
    и секретите (✏️)! Стар токен = тихо мълчание.

16. Settings → Actions → General → Workflow permissions →
    ✅ "Read and write permissions" → Save

──────────────────────────────────────────────
ЧАСТ 6 · ПУСКАНЕ
──────────────────────────────────────────────
17. Actions → AERO Gold Bot → Run workflow → Run
18. До ~2 мин: в групата идват картите (злато + сребро, ако има сигнал)
19. ПРОВЕРКА ЗА АВТОНОМНОСТ: след ~1 час в Actions трябва да има
    пускания "Scheduled" (пуснати САМИ). Има ли ги → готово ЗАВИНАГИ.

──────────────────────────────────────────────
АКО НЕЩО НЕ ДОЙДЕ — ДИАГНОЗА ЗА 10 СЕК
──────────────────────────────────────────────
Repo → папка live → live_journal.jsonl → последния ред:
  signal=SENT (200)        → пратено (виж групата)
  SEND_FAILED: 401         → грешен токен в секрета
  SEND_FAILED: 403         → ботът не е в групата / не е админ
  SEND_FAILED: 400         → грешен chat_id (минуса? -100…?)
  SKIPPED (без спам)       → същият сигнал вече е пратен (нормално)
Прати ми реда и оправяме за минута.

──────────────────────────────────────────────
КАКВО ПРАЩА БОТЪТ (автоматично, пон-пет)
──────────────────────────────────────────────

• 🎯 MA50/MA200 отскок-аларми (допълнителни сигнали).
• Без спам: само нови/сменени сигнали + събития.
• Времената са в СОФИЯ време.
• Седмично: качваш нов weekly_context.json (1 файл, аз ти го правя).
